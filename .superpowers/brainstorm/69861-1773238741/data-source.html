Where do inspiration events come from?

This shapes the entire architecture — pick the approach that fits best

A

Curated static calendar in the codebase

A hand-crafted list of UK hospitality-relevant dates (bank holidays, Valentine's Day, Bonfire Night, Six Nations, Wimbledon, etc.) maintained as structured data in the repo. The cron job filters this list for the next 180 days each month. Developers update the list annually. No external API calls.

Pros

  • No API dependency or cost
  • Full control over what's included
  • Works offline, deterministic
  • Very fast

Cons

  • Needs manual updates each year
  • Misses one-off events (elections, jubilees)
B

External API + curated fallback

Pulls UK bank holidays from the gov.uk API, supplements with a curated list for sporting/cultural events. The cron job merges both sources and deduplicates. Automatic for bank holidays; manual curation for the rest.

Pros

  • Bank holidays are always accurate (official source)
  • No manual date maintenance for bank holidays

Cons

  • External API dependency
  • Still need curated list for non-bank-holiday events
C

AI-generated list (Claude)

Each month, a cron job calls Claude to generate a list of UK hospitality-relevant events in the next 180 days. Claude knows about recurring events, sporting calendars, cultural moments, etc.

Pros

  • Can catch varied event types automatically
  • Minimal maintenance

Cons

  • Dates can hallucinate — unreliable for exact dates
  • API cost per run
  • Non-deterministic output

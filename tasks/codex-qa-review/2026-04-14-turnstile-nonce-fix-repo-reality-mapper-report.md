**Architecture Constraints For Later Reviewers**
- `middleware.ts` is the single nonce authority. Client components cannot call `headers()`, so any client-side `next/script` use under this CSP needs a server component to read `x-nonce` and thread it down as a prop.
- Public booking pages can arrive through two routes: direct `/l/[slug]` requests and `SHORT_LINK_HOST` rewrites from `/<slug>` to `/l/<slug>` at [middleware.ts:105](/Users/peterpitcher/Cursor/BARONS-BaronsHub/middleware.ts:105) to [middleware.ts:131](/Users/peterpitcher/Cursor/BARONS-BaronsHub/middleware.ts:131). Both paths rely on the same `x-nonce` contract.
- The app already uses `x-nonce` outside booking. `RootLayout` reads the same header at [layout.tsx:155](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/app/layout.tsx:155) to [layout.tsx:156](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/app/layout.tsx:156) and applies it to a `beforeInteractive` script at [layout.tsx:164](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/app/layout.tsx:164). This is broader CSP infrastructure, not a one-off booking pattern.
- Turnstile rendering is not fail-closed in this repo. `verifyTurnstile()` returns `true` when the token is missing, the secret is unset, the API is unavailable, or the fetch throws at [turnstile.ts:7](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/lib/turnstile.ts:7) to [turnstile.ts:18](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/lib/turnstile.ts:18) and [turnstile.ts:25](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/lib/turnstile.ts:25) to [turnstile.ts:36](/Users/peterpitcher/Cursor/BARONS-BaronsHub/src/lib/turnstile.ts:36). A broken widget can therefore be partially masked during functional testing.
- The booking flow has one extra coupling that auth forms do not: because it bypasses native form submission, the hidden `cf-turnstile-response` input must be manually extracted in the client component before the server action call. Any future refactor of `handleSubmit` or the widget DOM contract can break CAPTCHA verification independently of nonce plumbing.

## Observations

- **Audit logging is partial.** `logAuditEvent()` (the [[data-model#audit_log\|audit_log]] table) is called in `src/actions/users.ts` (user mutations), `src/lib/notifications.ts`, the `cascade-backfill` cron, and `src/lib/audit-log.ts`. Most other mutating action files (events, bookings, planning, venues, etc.) do **not** call it directly — a gap vs. the CLAUDE.md standard requiring audit logging on all mutations. See [[NOTES]] if tracking remediation.
- **All actions authenticate via `getCurrentUser()`**, then apply capability checks from `src/lib/roles.ts`.
- **`revalidatePath()`** is used consistently across mutating actions; no `revalidateTag`.
- **Feature flag** `EVENT_SAVE_USE_RPC` switches `events.ts` save/submit/propose between an atomic RPC path and a legacy fallback.

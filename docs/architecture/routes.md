## Auth Model

- **Middleware** (`middleware.ts`): intercepts all non-API requests; validates Supabase JWT + custom app-session-id cookie
- **Public paths**: `/login`, `/forgot-password`, `/reset-password`, `/auth/confirm`, `/unauthorized`, `/deactivated`, `/l` (landing pages)
- **API routes**: excluded from middleware matcher -- handle own auth via `requireWebsiteApiKey()` or `CRON_SECRET`
- **CSRF**: double-submit cookie pattern; token set in middleware, validated in server actions

## Middleware Auth Flow

`middleware.ts` applies to all routes except `/api/*`, `/_next/*`, and static assets.

1. **Domain check** â€” `l.baronspubs.com` rewrites slugs to `/l/[path]`; 8-char hex paths fall through to `[code]/route.ts`
2. **CSP nonce** â€” generated per request (16 bytes), forwarded as `x-nonce` header
3. **Security headers** â€” HSTS, X-Frame-Options DENY, CSP, Permissions-Policy
4. **Public path bypass** â€” `/login`, `/forgot-password`, `/reset-password`, `/auth/confirm`, `/unauthorized`, `/deactivated`, `/l/*`
5. **Supabase JWT** â€” `getUser()` validates session server-side (never `getSession()`)
6. **App session layer** â€” validates `app-session-id` cookie via `validateSession()`
7. **Session/user binding** â€” prevents session fixation (session.userId must match JWT user)
8. **Deactivation check** â€” redirects deactivated users to `/deactivated`
9. **CSRF token** â€” set if absent (JS-readable, SameSite=lax)

## Auth Flow

```
incoming request
  │
  ├─ l.baronspubs.com? ── yes ─▶ rewrite /slug -> /l/slug  (PUBLIC)
  │                                           │
  │                                           ▼
  │                                    src/app/l/[slug]/page.tsx
  │
  └─ baronspubs.com / primary domain
        │
        ▼
     middleware.ts
        │
        ├─ isPublicPath(pathname)? ── yes ─▶ set CSP + CSRF, return
        │
        ├─ has session cookie?
        │       │
        │       ├─ no  ─▶ redirect /login
        │       └─ yes ─▶ validateSession() + Supabase createServerClient
        │                   │
        │                   ├─ invalid    ─▶ clear cookie, redirect /login
        │                   ├─ deactivated─▶ redirect /deactivated (Supabase admin check on users.deactivated_at)
        │                   └─ valid      ─▶ set x-pathname, strip x-user-id, continue
        │
        ▼
     src/app/layout.tsx  (getCurrentUser → users row, role, venue_id)
        │
        ▼
     page.tsx / route.ts
        │
        ▼
     server action  →  src/lib/roles.ts  capability check
        │                    (isAdministrator, canManageEvents(role, venueId), ...)
        │
        ▼
     Supabase query (RLS enforced by current_user_role() + user.venue_id)
```

Files:
- Middleware: `middleware.ts`
- Session helper: `src/lib/auth/session.ts`
- Current user: `src/lib/users.ts`
- Role capabilities: `src/lib/roles.ts` (see [[overview#Auth Model Summary]])

`/api/*` is excluded from middleware (matcher: `/((?!api|_next/static|_next/image|favicon.ico).*)`). API handlers enforce their own auth:
- `/api/v1/*` — `requireWebsiteApiKey()` + `checkApiRateLimit()` in `src/lib/public-api/{auth,rate-limit}.ts`
- `/api/cron/*` — `CRON_SECRET` bearer
- `/api/webhooks/twilio-inbound` — Twilio signature validation
- `/api/auth/session-check` — Supabase SSR `getUser()`

## Auth Flow

End-to-end request flow (see [[overview#Auth Model]]):

```
Request
  → middleware.ts
      ├─ host == l.baronspubs.com? → rewrite /slug → /l/[slug]  (public landing, [[routes#Pages|/l/[slug]]])
      │                              └─ 8-hex code → fall through to [[routes#Short Link Handler|/[code]]]
      ├─ isPublicPath()? → set CSRF + security headers → return  (skip auth)
      ├─ supabase.auth.getUser()      → no user → redirect /login
      ├─ validateSessionWithRotation([[data-model#app_sessions|app_sessions]]) → fail → redirect /login (fail-closed)
      ├─ session.userId === user.id?  → mismatch → sign out + redirect (fixation guard)
      └─ users.deactivated_at set?    → redirect /deactivated
  → layout.tsx (getCurrentUser, skipped on auth/public pages)
  → page.tsx   (server component reads user + role)
  → server action ('use server')
      ├─ getCurrentUser()             → re-verify server-side
      ├─ src/lib/roles.ts capability  → e.g. canEditEvent / isAdministrator
      ├─ DB mutation (RLS enforced)
      ├─ logAuditEvent() (users.ts and select paths only)
      └─ revalidatePath()

/api/* routes bypass middleware (matcher excludes them) and self-authenticate:
  • /api/v1/*  → requireWebsiteApiKey() + checkApiRateLimit()
  • /api/cron/* → verifyCron() against CRON_SECRET
  • webhooks   → Stripe / Twilio signature verification
```

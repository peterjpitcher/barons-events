## Auth Model Summary

Three roles (deviation from workspace standard — approved):

| Role | Tier | Capability |
|------|------|------------|
| `administrator` | admin | Full platform access, all review decisions |
| `office_worker` | editor | Venue-scoped write if `venue_id` set; read-only otherwise |
| `executive` | viewer | Read-only across all events, planning, reporting |

Capability functions in `src/lib/roles.ts` take an optional `venueId` which switches `office_worker` between write and read-only. See [[relationships#Auth Flow]].

## Request Lifecycle

Key middleware responsibilities (`/Users/peterpitcher/Cursor/BARONS-BaronsHub/middleware.ts`):

- Deactivation check (redirects deactivated users to `/deactivated`)
- CSRF cookie issuance
- Matcher excludes `/api/*` — API routes do their own auth (bearer / cron / webhook signature)

## Public API

`/api/v1/*` is a rate-limited REST surface consumed by the BaronsPubs marketing site. All endpoints:
- Authenticate via `BARONSHUB_WEBSITE_API_KEY` (bearer or query param) — `src/lib/public-api/auth.ts`
- Rate-limit per-IP and per-API-key — `src/lib/public-api/rate-limit.ts`
- Return `{ success, data?, error? }` envelopes

See [[routes#Public API v1]] and [[relationships#Public API]].

## Cron Endpoints

All `/api/cron/*` routes are secured by a `CRON_SECRET` bearer token and invoked by Vercel Cron:
- `attachments-cleanup` — purge stale pending uploads
- `cascade-backfill` — backfill planning cascades from SOP updates
- `cleanup-auth` — prune expired sessions and login_attempts
- `expire-stale-approvals` — auto-reject stale review approvals
- `refresh-inspiration` — refresh `planning_inspiration_items`
- `sms-booking-driver`, `sms-post-event`, `sms-reminders` — Twilio dispatch jobs

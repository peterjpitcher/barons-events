## Auth Model

Auth is enforced in **`middleware.ts`** (root) for all non-`/api` routes, as a defense-in-depth pipeline. Full flow in [[relationships#Auth Flow]].

1. **Domain rewrite** — the `l.baronspubs.com` host rewrites slug paths to [[routes#Pages|/l/[slug]]] landing pages; 8-hex short links fall through to the [[routes#Short Link Handler|/[code]]] handler.
2. **Public path bypass** — `/login`, `/forgot-password`, `/reset-password`, `/auth/confirm`, `/unauthorized`, `/deactivated`, `/l/*`, and static assets skip the auth gate.
3. **Supabase JWT check** — `supabase.auth.getUser()` (never `getSession()`); no user → redirect `/login`.
4. **Custom app session** — `validateSessionWithRotation()` against an app-session-id cookie ([[data-model#app_sessions|app_sessions]]); fail-closed on any error; rotates token; verifies `session.userId === user.id` (session-fixation guard).
5. **Deactivation check** — service-role lookup on `users.deactivated_at` → redirect `/deactivated`.
6. **Security headers** — CSP (production only), nonce, CSRF cookie, `Permissions-Policy`, no-store, applied to every response.

**`/api/*` routes are excluded from middleware** (`matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"]`) and authenticate themselves:
- **Public API** (`/api/v1/*`) → `requireWebsiteApiKey()` + `checkApiRateLimit()` (Bearer token / query param).
- **Cron** (`/api/cron/*`) → `verifyCron()` against `CRON_SECRET`.
- **Webhooks** (`stripe`, `twilio-inbound`) → provider signature verification.

### Custom Role Model

Roles live in `public.users.role` (not Supabase `app_metadata`), with capability helpers in `src/lib/roles.ts`. `venue_id` acts as a capability switch for `office_worker`.

| Role | Tier | Capability |
|------|------|-----------|
| `administrator` | admin | Full platform access, user management, review/approve events |
| `office_worker` | editor | Venue-scoped write (with `venue_id`) or global read + event proposals (without) |
| `executive` | viewer | Read-only across events, planning, reporting |

Capability functions: `isAdministrator()`, `canProposeEvents()`, `canEditEvent()`, `canReviewEvents()`, `canViewEvents()`, `canManageBookings()`, `canManageCustomers()`, `canManageArtists()`, `canCreateDebriefs()`, `canSubmitDebriefForEvent()`, `canEditDebrief()`, plus planning/SOP/link/attachment helpers.

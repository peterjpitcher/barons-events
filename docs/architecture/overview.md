## Auth Model

Authentication is layered:

1. **Supabase Auth** issues a JWT cookie when a user signs in.
2. **`middleware.ts`** validates the JWT via `supabase.auth.getUser()` on every non-public route, then validates the custom app-session cookie via `validateSession(appSessionId)`.
3. **Public paths** (login, forgot/reset password, unauthorized, deactivated, `/l/*` short links) bypass the auth gate.
4. **`/api/*` routes are excluded from middleware** (matcher config) — they implement their own auth via `requireWebsiteApiKey()` (public API), `verifyCronSecret()` (cron), or `supabase.auth.getUser()` (session-check).
5. **Server actions** in [[server-actions]] re-verify auth server-side via `getCurrentUser()` from `src/lib/auth`.
6. **RBAC** is enforced via capability functions in `src/lib/roles.ts` — see [[#Role Model]].
